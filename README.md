# QAzando
# 🚀 QAZANDO - QA Automation Portfolio

## 📌 Sobre o Projeto

Projeto de estudos em **Quality Assurance (QA)** utilizando **testes automatizados, documentação de qualidade e Inteligência Artificial aplicada à automação de testes**, desenvolvido durante o curso profissionalizante da **QAzando**.

O objetivo deste projeto é aplicar conceitos e boas práticas do mercado de QA, simulando um ambiente real de desenvolvimento e validação de software.

---

## 🎯 Objetivos

Este projeto tem como objetivo praticar e demonstrar conhecimentos em:

- Planejamento de testes;
- Criação e documentação de casos de teste;
- Registro, análise e acompanhamento de bugs;
- Testes Web;
- Testes de API;
- Testes Mobile;
- Automação de testes;
- Boas práticas de qualidade de software;
- Utilização de Inteligência Artificial como apoio na criação e manutenção de testes automatizados.

---

## 🛠️ Tecnologias e Ferramentas

| Tecnologia/Ferramenta | Utilização |
|---|---|
| **Cypress** | Automação de testes End-to-End (E2E) para aplicações Web |
| **Playwright** | Automação de testes Web e validação de diferentes navegadores |
| **Postman** | Criação e execução de testes de APIs REST |
| **Maestro** | Automação e testes de aplicações Mobile |
| **Git/GitHub** | Versionamento de código e gerenciamento do projeto |
| **IA Generativa** | Auxílio na criação, análise e otimização de testes automatizados |

## 🧪 Tipos de Testes Aplicados

### Testes Web
Validação de funcionalidades da aplicação através de testes automatizados:

- Fluxos de autenticação;
- Cadastro de usuários;
- Validação de mensagens;
- Cenários positivos e negativos;
- Testes de regressão.

---

### Testes de API

Validação de serviços REST:

- Requisições HTTP;
- Status codes;
- Headers;
- Payloads;
- Validação de respostas;
- Cenários de sucesso e erro.

---

### Testes Mobile

Automação de fluxos em dispositivos móveis:

- Navegação entre telas;
- Validação de componentes;
- Testes funcionais;
- Cenários críticos da aplicação.

---

## 📝 Documentação de QA

A documentação do projeto contempla:

### Casos de Teste

Registro contendo:

- Identificação do cenário;
- Pré-condições;
- Passos para execução;
- Resultado esperado;
- Resultado obtido;
- Status do teste.

### Relatórios de Bugs

Os bugs documentados possuem:

- Título;
- Descrição;
- Classificação;
- Severidade;
- Prioridade;
- Ambiente;
- Passos para reprodução;
- Evidências;
- Resultado esperado x resultado encontrado.

---

## 🤖 Uso de Inteligência Artificial

A Inteligência Artificial é utilizada como ferramenta de apoio para:

- Geração e melhoria de cenários de testes;
- Análise de casos de teste;
- Criação de scripts automatizados;
- Sugestão de melhorias na cobertura de testes;
- Auxílio na documentação técnica.

## ▶️ Execução dos Testes

### Cypress

Instalar dependências:

```bash
npm install
```

Executar interface do Cypress:

```bash
npx cypress open
```

Executar testes em modo automático:

```bash
npx cypress run
```
